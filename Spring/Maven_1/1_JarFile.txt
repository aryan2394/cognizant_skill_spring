Imagine this situation

Suppose you made a React project.

Before deploying it, you run:

npm run build

After that, React creates a build folder.

Inside that folder are all the HTML, CSS, and JavaScript files packed and optimized so the browser can run your website.

👉 That build folder is the final version of your React app.

Java has something similar

When you write Java code, you have lots of files.

Example:

Main.java
Student.java
Teacher.java
Utils.java
Database.java

Java first compiles them into

Main.class
Student.class
Teacher.class
Utils.class
Database.class

These .class files are machine-readable bytecode.

Now imagine you have 100 class files.

Instead of sending all 100 files separately, Java puts them into one file.

That file is called a JAR file.

JAR = Java ARchive

Think of it like a ZIP file

Suppose you have

Photo1.jpg
Photo2.jpg
Photo3.jpg

You compress them into

Photos.zip

Similarly,

Main.class
Student.class
Teacher.class

becomes

project.jar

A JAR file is basically a special ZIP file for Java programs.

What's inside a JAR?

Suppose your project looks like this

Project

Main.class
Student.class
Teacher.class

images/
logo.png

config.properties

After creating a JAR

project.jar

Inside it you'll find

Main.class
Student.class
Teacher.class

logo.png

config.properties

Everything is packed together.

Why do we need a JAR?

Imagine you built a calculator application.

Without a JAR

Calculator/

Main.class
Button.class
Math.class
Config.class
Image.png

If you want to send it to a friend, you'd need to send the whole folder.

With a JAR

Calculator.jar

You only send one file.
