Dependency Injection:

Agar ek class ko dusri class ki functionality use karni hai, to us class ka object khud new karke create nahi karna chahiye. 
Object kisi aur se receive karna chahiye (constructor, setter ya Spring Container ke through). 
Isse classes loosely coupled rehti hain aur code ko maintain aur modify karna easy ho jata hai.

Step 1: Problem

Suppose tumhare paas do classes hain.

class Engine {

    public void start() {
        System.out.println("Engine Started");
    }
}
class Car {

    Engine engine = new Engine();

    public void drive() {
        engine.start();
        System.out.println("Car is Running");
    }
}

Question:

👉 Engine object kisne banaya?

Answer:

Engine engine = new Engine();

Car ne.

Yahi problem hai.

Car ka kaam Engine banana nahi, balki Engine use karna hai.

Ye Tight Coupling hai.


Step 3: Solution

Car object khud mat banao.

Koi aur bana ke de.

class Car {

    Engine engine;

    public Car(Engine engine) {
        this.engine = engine;
    }

    public void drive() {
        engine.start();
    }
}

Ab Car ne kahin bhi

new Engine()

nahi likha.
