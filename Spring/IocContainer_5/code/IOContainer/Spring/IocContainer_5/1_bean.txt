Sabse pehle ek normal Java example

Maan lo tumhare paas ek class hai:

class Car {
    public void start() {
        System.out.println("Car Started");
    }
}

Ab object banane ke liye tum likhte ho:

Car car = new Car();
car.start();

Yahan

Car → Class
car → Object

Object tumne khud banaya.

Spring mein kya hota hai?

Spring bolta hai:

Object tum mat banao. Main banaunga.

Isliye tum new kam use karte ho.

Spring object create karta hai, usko manage karta hai aur jab zarurat hoti hai tab de deta hai.

Spring ke dwara manage kiya gaya object hi Bean kehlata hai.

Definition

A Bean is simply a Java object that is created, configured, and managed by the Spring IoC Container.

Is definition ko todte hain.

1. Created

Spring object banata hai.

Normal Java

Car car = new Car();

Spring

Spring Container
       |
       |
       ▼
Creates Car object

Tumhe new Car() nahi likhna padta.

2. Configured

Spring us object ki settings bhi kar deta hai.

Example

class Car {

    Engine engine;

}

Spring automatically engine bhi laga sakta hai.

Car
 |
Engine

Tumhe manually

car.setEngine(engine);

nahi likhna padta.

3. Managed

Ye sabse important point hai.

Managed matlab Spring object ka dhyan rakhta hai.

Jaise

kab banana hai
kitni baar banana hai
kisko dena hai
kab destroy karna hai

Sab Spring dekhta hai.
