example:class OrderService,PaymentService hai 

agar class OrderService ko PaymentService ka object chahiye apne mein and PaymentService is independent matlab agar wo kisis pe depened nahi hai toh 
IOC Container bahut smart hai pahle wo PaymentService ka beans banayega phir usko inject kar dega in OrderService
using contructor,setter or field injection

        System.out.println("order details");

class PaymentService:

@Component
public class PaymentService {
    private OrderService orderService;
    public PaymentService(OrderService orderService)
    {
        this.orderService=orderService;
    }
    public void pay()
    {
        System.out.println("payment done");
        orderService.getDetails();
    }
}

LEKIN MAIN QUESTION HAI KI AAP circular dependency AANE HI KYON DE RAHE HO iska solution kya hai ?

matlab circular dependency aaya hi kyon uske peeche reason hai kyonki orderService ko ek method call krana tha or usko ek object chahiye tha so that's why humne uske objevt ko inject kiya 
lekin aapne PaymentService mein OrderService ka object kyon banaya isliye abnaya hai kyonki humein ek orderService ka ek method call karna hai 

Practically:dekhe toh order karne se pahle usko payment ki jarrorat hai and payment karne ke liye usko order karne ki jarrorat hai toh aapko usko 
orderService mein toh pay() function chalana hi padega hi kyonki uske bina toh order complete ho hi nahi sakta and aapko pay() mein OrderDetails ke liye 
orderService chahiye hi hai kyonki bina payment ke kaun sa order complete hoga lekin 

PaymentService ne bola ki mujhe pay karne ke baad ab OrderDetails() function chahiye jo ki object ka lekin bhai tumhe kya matlab hai ki OrderDetails() kya hai tum apana kaam karo n
follow the SOLID prinCIPLE(single resposnibility prinCIPLE) toh iska solution ye hai ki aap payment kar do and orderService payment hone ke baad apne method ko khud se call kar le n dusre ko kyon bol raha hai ki aap payment ke baad OrderDetails() call kar lena
