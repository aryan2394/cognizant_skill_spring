Throwable is the root class of Java's exception hierarchy.
It has two direct subclasses:
Error в†’ Serious JVM/system errors (generally not handled).
Exception в†’ Conditions that applications can handle.
Exception is divided into:
Checked Exceptions в†’ Must be handled (try-catch) or declared using throws.
Unchecked Exceptions (RuntimeException) в†’ Handling is optional; they usually occur because of programming mistakes.

Easy Rule (Interview Point в­ђ)

Multiple catch blocks are always written from the most specific exception (child) to the most general exception (parent).

Reason:

JVM first matching catch execute karti hai.
Agar parent pehle likh diya, to child exception kabhi us tak pahunch hi nahi payegi.
Isliye compiler usse unreachable code maan kar compile-time error de deta hai.

Example
try {
    int a = 5;
    int b = 0;

    System.out.println(a / b);
}
catch (ArithmeticException e) {
    System.out.println("Divide by zero is not allowed");
}
catch (RuntimeException e) {
    System.out.println("Runtime Exception");
}
catch (Exception e) {
    System.out.println("Generic Exception");
}
